The HTML 2.0 specification includes both machine-readable public text -- SGML "code" if you will -- and human-readable text. The public text includes the DTD, an SGML declaration, and a version of the ISO Added Latin 1 entity set.

Changes in the November 30 Revision

  • Format to <72 chars for RFC style.
  • Fix some parameter entities that were illegal, but unflagged by SGMLS.
  • Remove quiryky newline stuff in SGML declaration

Changes in the November 30 Revision

  • Integrated ICADD SDA attributes and APPINFO (Thanks to Jeff Suttor)
  • Updated catalog etc. with new FPIs (formal public identifiers).
    Old format: "-//IETF//DTD HTML//EN//2.0"
    New format: "-//IETF//DTD 2.0 HTML//EN"
  • Changed FUNCTION character syntax: uncommented the real one, commented out the hack version.
  • Added FPI's and public text for "Strict" versions of the DTD (e.g. "-//IETF//DTD HTML Strict Level 1//EN" maps to html-1s.dtd)
  • Changed FPI used to reference ISO latin 1 entities.
  • Changed a bunch of + occurence indicators to * for clarity
  • Changed some parameter entities to comply with SGML tokenization rules.

Changes in the November 15 Revision

  • Added one-line comment for each element and attribute in the DTD.
    Removed some other comments
  • ALT is no longer required in level 0: some images are completely inessential.
  • Changed DL content model to (DT | DD)+
  • ADDRESS content model is always (%text|P)*
  • FORM ACTION attribute is #IMPLIED (defaults to address of document)
  • Removed remaining references to World-Wide Web

See also: changelog of html.dtd from 1992/12/03 thru 1994/11/15
